接口说明_R9

1.概述

1.1特性

R9产品是在R8N的基础上升级得到,产品更稳定,算法更优,主要有如下特性:

  1. 新增体温监测,新增血氧监测；

  2. 调优心率传感器；

  3. 调优GPS定位稳定性.

R9产品主要包含三部分接口：

  • 基于MQTT的NB-IoT接口；

  • 基于HEX的BLE接口;

  • 基于UDP的UWB接口.

1.2默认

  1. 刷新固件后R9初始为室内UWB工作模式,开始以1s的间隔定位输出JSON,静息1min后以30s的间隔定位输出JSON;

  2. 若到室外无UWB基站覆盖的范围,R9切换到室外高频次上传模式;

  3. 整个过程体征传感器以红灯方式周期性工作;

  4. 开发接口可根据需要自行设置.

2.NB-IoT接口

2.1描述

当R9不在UWB基站覆盖范围内时,一定时间后R9自动切换到室外模式,并以高频次上报数据到MQTT服务端(实测最快5S/次).注意体核温度是关于体表温度和环境温度的二元二次方程,终端计算能力有限,因此给出参考函数关系:

Z=0.0337y^2-0.545y+1.7088x-0.0519xy+17.626

其中Z表示体核温度、y表示体表温度、x表示环境温度。

2.2定位上行

定位上报分连续上报和日常上报.

示例： {"report_type":"outdoorlocation","time":"20180910T120530","lat":"3213.098739","log":"11837.112425","tag_id":"D402117B9639","tag_type":"04A1","battery":"85","step":"8500","heart_rate":"118","temperature_skin":"31.3","temperature_env":"26.6","spo2":"98","wear_status":"1","blood_press_h":"115","blood_press_l":"70"}

2.3 SOS上行

长按红色按键3秒以上即可向平台发送SOS数据包.

示例： {"report_type":"sos","time":"20000101T004329","tag_id":"FE27DFCC873C","tag_type":"04A1","msg_id":"946687409"}

2.4 时间校准

用于设置终端的当前时间，服务端发现上报时间出现较大偏差时可在收到上报数据后立即下行指令,重新校正时间. 1.请求

示例： {"setTime":""1566197383,8cd49500fffe,20200926T151830"} 2.应答

示例： {"ackSetTime":"1614018411,C271D9F84236,20210222T182651"}

2.5 开关设置

用于设置终端的主要外设开关状态,NB-IoT要作为命令交互(即使关闭NB-IoT，NB-IoT的系统日常上报是无法关闭)目前包含:室内外自动切换是否开启;GPS是否开启;UWB的开关与快速定位间隔(正常的快速定位过程中保持静息,默认1分钟后切换到慢速定位);慢速定位间隔;心率开关.

  • 线下测试方式:长按红色SOS按键3秒,此时SOS信息上报,MQTT端收到JSON定位数据后,立刻设置外设状态,设置后会有应答指令.若因NB-IoT网络问题,稍后可以再试.

  • 线上使用方式: 任何时刻MQTT端收到JSON定位数据后,立刻设置外设状态,设置后会有应答指令.

1.设置

外设状态说明(需按顺序设置):

  1. NB-IoT状态:1-关闭;2-开启用户日常上报(后紧跟时间);3-开启连续上报.

  2. 室内外自动切换状态:0-关闭;1-开启.

  3. GPS开关状态:0-关闭;1-开启.

  4. UWB开关状态:0-关闭;1-开启(后紧跟时间).

  5. UWB慢速时间.

  6. 心率开关与周期性开启时间(工作周期4min,默认不可修改)1-关闭;2-一直开启;3-周期性开启.

例如,NB-IoT开启且15min上报一次(9000),允许室内到室外切换,开启GPS,UWB开启并且1s快速定位一次;UWB慢速30s定位一次;心率开启并且每隔40s采集4min. 示例: {"setPeripherals":"1566197383,8cd49500fffe,2-9000,1,1,1-10,300,3-400,"} 2.应答

例如,NB-IoT开启且15min上报一次(9000),不允许室内到室外切换,关闭GPS,UWB开启并且1s快速定位一次;UWB慢速30s定位一次;心率开启并且每隔40s采集4min. 示例: {"setPeripherals":"1566197383,8cd49500fffe,1-9000,0,0,1-10,300,1-400,"}

2.4 IP设置

用于设置终端的MQTT的ip和port，注意确保ip地址有效,否则手表变砖. 1.请求

示例： {"setIp":"1566197383,8cd49500fffe,112.113.114.115,1883,"} 2.应答

示例： {"ackSetIp":"1614018411,C271D9F84236,112.113.114.115,1883,"}

3.BLE接口

3.1广播接口

R9广播有 2 种：一种是可连接的广播，一种是不可连接的广播。当 手环处于未连接状态时，将广播可连接的广播数据； 当手环已经被连接后，将会广播不可连接的广播数据。可连接的广播数据：包含主要的 UUID 和设备名字，设备类型和型号，MAC 地址，用户自定义数据域。 不可连接的广播数据：包含 MAC 地址（蓝牙广播标准），用户自定义数据域（包含在广播数据中）。其中，UUID、用户自定义数据包含在广播数据中；而设备类型和型号、MAC 地址，包含在扫描响应数据中。即B8字段位于广播数据内，B6、B7字段位于扫描应答数据里。广播间隔为1秒。蓝牙标准广播数据格式为： length+flag+data。 我们规定的自定义数据域位于蓝牙标准广播中 flag=0xff 后面的 data 里。 自定义数据域数据格式：package_ID+data，具体数据协议如下表：

Package_ID为0xB8的自定义数据域1内容格式：

字节1：心率值(bpm)

字节2：步数低字节bit[0:7]

字节3：步数中字节bit[8:15]

字节4：步数高字节bit[16:23]

字节5：拆带与否：0已拆，1未拆

字节6：动静状态：0动，1静

字节7：血氧浓度(%)

字节8：环境温度整数部分(℃)

字节9：环境温度小数部分(℃)

字节10：体表温度整数部分(℃)

字节11：体表温小数部分(℃)

字节12：收缩压(mmHg)

字节13：舒张压(mmHg)

字节14：保留

字节15：保留

字节16：保留

字节17：电池电量（百分比）

字节18：异或校验值（0xB8至字节18的异或校验值），尚未启用

4.UWB接口

接口数据输出是由算法单元以UDP形式输出，包含多个维度的信息。

示例： {"indoorLocation":{"additionMessage":{"band":"on","battery":"10","heart_rate":"76","silent":"false","step":"0","temperature_env":"28.0","temperature_skin":"29.5","spo2":"98","wear_status":"1","blood_press_h":"115","blood_press_l":"70"}},"coordx":"0.84","coordy":"11.19","coordz":"NULL","dimension":"0","locationTime":"20210128T112801","panid":"b1","qaulity":"3","rssi":"-70","tagID":"D170F5C9CB92"}}
